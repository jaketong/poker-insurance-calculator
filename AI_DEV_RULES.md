# AI_DEV_RULES.md

本文件用于固定扑克保险工具项目的业务规则、禁止范围和验收标准。

所有 Cursor / Codex / AI Agent 在修改本项目时，必须遵守本文件。
如果任务指令与本文件冲突，必须停止并向用户确认。

---

## 1. 项目定位

本项目是手机版线下扑克保险计算工具。

当前定位：

- 手机网页
- 一个链接打开即用
- 不登录
- 不安装 App
- 无后端
- 不保存用户输入
- 面向线下牌局
- 支持德州扑克、奥马哈、短牌三种独立工具
- 只做两家 All-in 保险计算
- 用户手动指定领先方
- 结果页要适合截图转发
- 支持复制结果文本

---

## 2. 当前明确不做

未经用户明确确认，禁止新增以下功能：

- 登录
- 注册
- 用户系统
- 后端服务
- 数据库
- 保存用户输入
- 多人 All-in
- 边池
- 自动判断领先方
- GTO 分析
- 起手牌助手
- 历史记录
- 账号同步
- 支付功能
- 复杂后台
- 自动部署
- 多 Agent 并行修改

---

## 3. 通用业务规则

所有玩法必须遵守：

- 只计算两家 All-in 场景
- 领先方由用户手动指定
- 工具计算的是落后方反超领先方的 outs
- 保险购买方是当前领先方
- 保险命中概率基于反超 outs
- 不自动判断谁领先
- 不处理多人底池
- 不处理边池
- 不加入 GTO 决策
- 不加入平台推荐逻辑

---

## 4. 德州扑克规则

德州扑克工具必须保持：

- 使用标准 52 张牌
- 每名玩家 2 张手牌
- 公共牌按德州扑克规则处理
- 支持两家 All-in 保险计算
- 用户手动指定领先方
- 根据落后方反超领先方的 outs 计算保险相关结果

德州扑克赔率表规则：

- 2 outs 的赔率为 15 倍
- 赔率表属于业务规则
- 未经用户明确确认，不得修改赔率表

---

## 5. 奥马哈规则

奥马哈工具必须保持：

- 使用标准 52 张牌
- 每名玩家 4 张手牌
- 必须严格使用 2 张手牌 + 3 张公共牌成牌
- 不能复用德州扑克成牌逻辑
- 支持两家 All-in 保险计算
- 用户手动指定领先方
- 根据落后方反超领先方的 outs 计算保险相关结果

奥马哈高风险点：

- 不允许把 1 张手牌 + 4 张公共牌算作有效牌型
- 不允许把 3 张手牌 + 2 张公共牌算作有效牌型
- 不允许把 4 张手牌直接参与最终成牌
- 不允许为了省事复用德州扑克判断逻辑

---

## 6. 短牌规则

短牌工具必须保持：

- 使用 36 张牌
- 牌面范围为 6、7、8、9、T、J、Q、K、A
- 不使用 2、3、4、5
- 支持两家 All-in 保险计算
- 用户手动指定领先方
- 根据落后方反超领先方的 outs 计算保险相关结果

短牌默认规则：

- A6789 是顺子
- 同花大于葫芦
- 顺子大于三条

短牌高风险点：

- 不允许把短牌当作标准 52 张牌德州扑克处理
- 不允许加入 2、3、4、5
- 不允许擅自改变短牌牌型大小顺序
- 不允许擅自改变 A6789 顺子规则

---

## 7. 赔率表修改规则

赔率表属于业务规则，不属于普通代码优化。

未经用户明确确认，禁止修改：

- 德州扑克赔率表
- 奥马哈赔率表
- 短牌赔率表
- 赔率显示格式背后的实际倍率
- 任何 outs 对应赔率

如果任务涉及赔率表，必须停止并返回：

1. 当前准备修改哪张赔率表；
2. 为什么需要修改；
3. 修改前后对照；
4. 是否会影响已有计算；
5. 需要用户确认的问题。

---

## 8. 算法修改规则

算法属于高风险区域。

未经用户明确确认，禁止修改：

- outs 计算逻辑
- 反超牌枚举逻辑
- 平分牌枚举逻辑
- 胜负判断逻辑
- 命中概率计算逻辑
- 买保本计算逻辑
- 买满池计算逻辑
- 德州扑克成牌逻辑
- 奥马哈成牌逻辑
- 短牌成牌逻辑

如果任务必须修改算法，必须先停止，不得直接改。

算法任务开始前必须提供：

1. 要修改的具体算法点；
2. 为什么必须修改；
3. 会影响哪种玩法；
4. 修改前后预期结果；
5. 至少 1 个验收用例；
6. 是否影响赔率表；
7. 是否需要用户确认业务规则。

---

## 9. UI / 文案修改规则

以下属于低风险任务，可以在明确任务下执行：

- 错误提示优化
- 页面说明文字优化

但 UI / 文案任务必须遵守：

- 不改算法
- 不改赔率表
- 不改玩法规则
- 不改依赖
- 不改部署配置
- 不新增后端
- 不新增数据库
- 不新增登录
- 不顺手重构无关代码

---

## 10. 复制结果文本规则

复制结果文本应尽量包含：

- 游戏类型
- 领先方
- 落后方
- 反超 outs
- 保险命中概率
- 保险赔率
- 建议买保
- 买满池参考
- 必要时显示反超牌
- 必要时显示平分牌
- 简短免责声明

建议免责声明：

```text
说明：结果仅供线下牌局保险计算参考，请以实际牌局规则为准。
```

## 11. GitHub 与 Netlify 发布规则

1. GitHub 仓库地址为：  
   https://github.com/jaketong/poker-insurance-calculator

2. Netlify 当前已连接 GitHub 仓库，并从 master 分支自动部署。

3. push 到 master 会触发 Netlify 自动部署生产站点。

4. 未经用户明确确认，不得执行 git push。

5. 未经用户明确确认，不得手动触发 Netlify deploy。

6. 每次准备 push 前必须完成：

   - git status --short
   - 确认工作区干净或只包含预期改动
   - 确认没有 dist、node_modules、.env、日志、临时文件、数据库文件
   - 确认没有 package.json、lock 文件、部署配置的意外变更

7. 每次 push 后必须检查 Netlify Deploys 页面：

   - 状态必须为 Published
   - 部署提交必须与本次 push 的 commit 一致
   - 如果部署失败，只查看部署日志，不得扩大修改范围

8. Netlify 配置属于高风险项，未经用户确认不得修改：

   - Build command
   - Publish directory
   - Branch to deploy
   - Environment variables
   - Domain settings
   - Deploy hooks
   - Plugins

9. 当前标准发布流程为：

   Cursor 修改代码  
   → ChatGPT 验收  
   → Codex 审查  
   → 用户确认提交  
   → git commit  
   → 用户确认 push  
   → git push origin master  
   → Netlify 自动部署  
   → 检查 Published
