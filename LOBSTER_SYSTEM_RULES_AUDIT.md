# LOBSTER_SYSTEM_RULES_AUDIT.md

## 1. 文件目的

本文件用于审计 **V1.1「远程开发流程协议化」** 是否已经把角色分工、高风险边界、标准流水线写成**可对照、可验收、可复盘**的固定规则。

- **协议化**：各工具做什么、不做什么，有文字约定。  
- **可复用**：换一轮任务仍按同一套流程走。  
- **可验收**：每条规则对应「是否做到」的勾选或说明。  
- **可复盘**：出问题时可对照本表查环节，而不是凭记忆。

本文件**不**定义扑克保险业务算法，**不**替代 `AI_DEV_RULES.md` / `POKER_INSURANCE_ACCEPTANCE_CASES.md`；仅描述**龙虾式远程 AI 开发系统**的总控与协作规则是否已落地。

---

## 2. 当前系统总目标

把 **ChatGPT + Cursor + Codex + GitHub + Netlify + 腾讯云服务器** 固定成一套**远程 AI 开发执行系统**：

- 需求与风险在总控侧收敛；  
- 代码变更在 Cursor 侧执行并留痕；  
- 越界与质量在 Codex 侧审查；  
- 版本与回滚在 GitHub；  
- 前端站点在 Netlify（由 GitHub 触发）；  
- 服务器作为**后期远程控制中心**，V1.1 阶段**不**强求自动化主角地位。

## 3. 工具职责审计表

| 工具 | 当前职责 | 禁止事项 | 是否已固定 | 备注 |
|------|----------|----------|------------|------|
| ChatGPT | 总控、方案判断、边界控制、任务压缩、验收、复盘 | 代替 Cursor 直接改仓库；未经流程擅自要求 push/部署 | 是（文档与对话约定） | 以任务书、验收清单、复盘沉淀为主 |
| Cursor | 主执行：按明确任务改代码、跑构建/检查、返回结果 | 擅自扩大范围；擅自改算法/赔率/依赖/部署；擅自 push | 是（AGENTS / AI_DEV_RULES 等） | 执行粒度由任务书约束 |
| Codex | 审查员：检查改动是否越界、是否可合并思维 | 替代产品决策；与 Cursor 同时无约束改同一分支 | 是（审查节点约定） | 高风险变更必经审查 |

| Netlify | 部署器：仅通过 GitHub 连接自动构建发布 | 手动随意改生产配置；绕过 Git 的「神秘部署」 | 是（连接 GitHub 后） | 配置变更属高风险，须用户确认 |
| 腾讯云服务器 | 后期远程控制中心、运维与扩展承载 | V1.1 强依赖服务器自动化；密钥进对话明文 | 部分（阶段策略） | 当前阶段**不作为第一阶段主角** |

## 4. 高风险停止规则审计表

| 风险项 | 当前规则 | 是否必须用户确认 | 当前是否允许 | 备注 |
|--------|----------|------------------|--------------|------|
| package.json / lock 文件修改 | 非用户明确要求，AI 不主动改 | 是 | 否（默认） | 依赖变更影响面大 |
| 新增依赖 | 禁止未经确认的 `npm install` / 加包 | 是 | 否（默认） | 与 lock、构建链路绑定 |
| 核心算法修改 | 须业务确认 + 验收用例 | 是 | 仅用户明确任务时 | 扑克保险项目见专项规则文档 |
| 赔率表修改 | 须用户明确确认 | 是 | 否（默认） | 属业务规则非普通 refactor |
| Netlify 配置修改 | Build / Publish / 分支 / 环境变量等须用户确认 | 是 | 否（默认） | 误配可导致生产不可用 |
| GitHub Actions 修改 | 视同部署/CI 高风险 | 是 | 否（默认） | 可与密钥、令牌交互 |
| .env / 密钥修改 | 禁止进仓库与明文传递 | 是 | 否（默认） | 仅用户本地/托管面板操作 |
| 数据库 / 后端 / 登录 | V1.1 扑克保险主线不做 | 是 | 否 | 与本工具定位一致 |
| 删除文件 | 须任务明确；禁止顺手删 | 视任务 | 仅任务允许时 | 防止「清理」扩大化 |
| git push | 仅用户明确允许后由人或受控流程执行 | 是 | AI 默认禁止 | 避免误发版 |
| 生产部署 | 与 push 联动；禁止 AI 手动乱点生产 | 是 | 仅 Netlify 自动链路 | 人工验收在发布后 |
| 腾讯云服务器自动化 | V1.1 不做为主流程 | 是 | 否 | 与「阶段 12」等边界一致 |
| 多 Agent 并行 | 禁止无协调多写同一分支 | 是 | 否 | 降低冲突与责任不清 |

## 5. 标准开发流程审计

以下为 **V1.1 推荐标准流程**（与 `AI_DEV_RULES.md` 中 GitHub + Netlify 发布规则一致部分应互相印证）：

1. 用户提出任务  
2. ChatGPT 判断目标、阶段、风险等级  
3. ChatGPT 生成 Cursor 执行任务（范围、禁止项、成功标准写清）  
4. Cursor 执行并返回结果（含 `git status`、构建等约定输出）  
5. ChatGPT 初步验收  
6. 必要时 Codex 审查  
7. ChatGPT 判断是否可提交（文件范围、风险项对照）  
8. 用户确认 **commit** / **push**  
9. GitHub 保存版本  
10. Netlify 自动部署（监听分支）  
11. 用户线上人工验收  
12. ChatGPT 复盘并沉淀规则（更新 AGENTS / AI_DEV_RULES / 本审计表等）

## 6. 当前阶段不做清单（V1.1）

当前 V1.1 **明确不做**：

- 不做服务器自动化（不作为主线默认步骤）  
- 不接 n8n  
- 不做任务面板  
- 不做多 Agent 并行改同一仓库  
- 不让 AI **自动** push  
- 不让 AI **自动**部署生产  
- 不改 package 配置（除非用户单独立项并确认）  
- 不处理密钥自动化  
- 不买新服务器
