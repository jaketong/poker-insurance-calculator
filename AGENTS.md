# AGENTS.md

本文件是给 Cursor / Codex / AI 编程助手读取的项目执行规则。

## 项目定位

这是一个手机版线下扑克保险计算工具。

项目特点：
- 手机网页，一个链接打开即用
- 不登录
- 不安装 App
- 无后端
- 不保存用户输入
- 面向线下牌局
- 支持德州扑克、奥马哈、短牌三种独立工具
- 当前只做两家 All-in 保险计算

## 工具职责

ChatGPT 是大脑，负责：
- 判断任务是否值得做
- 拆分任务
- 控制边界
- 生成执行指令
- 验收结果
- 复盘沉淀规则

Cursor / Codex 是执行工具，只负责：
- 按明确任务修改代码
- 运行指定检查命令
- 返回修改结果
- 不做产品方案判断
- 不自由发挥
- 不扩大任务范围

## 当前开发原则

1. 主线优先：先跑通，再稳定，再优化，再美化。
2. 能合并的步骤就合并，避免过度拆分。
3. 不写无用文档，不浪费开发额度。
4. 修改后必须验收，验收通过再提交。
5. 提交前后必须检查 git status。
6. 失败时只修失败点，不扩大范围。

7. 不新增后端。
8. 不新增数据库。
9. 不新增登录功能。
10. 不保存用户输入。
11. 不删除文件。
12. 不 git push。
13. 不部署。
14. 不自动提交。
15. 不做多人 All-in。
16. 不做边池。
17. 不做 GTO。
18. 不做起手牌助手。
19. 不自动判断领先方。

## 扑克业务规则

德州扑克：
- 用户手动指定领先方。
- 工具计算落后方反超 outs、保险命中概率、买保本、买满池。
- 德州扑克赔率表中 2 outs 为 15 倍。
- 未经用户确认，不得修改赔率表。

奥马哈：
- 必须严格使用 2 张手牌 + 3 张公共牌组成牌型。
- 不得复用德州扑克的牌型判断规则。
- 未经用户确认，不得修改奥马哈规则和赔率表。

短牌：
- 使用 36 张牌，牌面为 6-A。
- 默认 A6789 是顺子。
- 默认同花大于葫芦。
- 默认顺子大于三条。
- 未经用户确认，不得修改短牌规则和赔率表。

## 高风险停止规则

如果任务涉及以下内容，必须停止并向用户确认：

1. 修改核心算法。
2. 修改 outs 计算逻辑。
3. 修改胜率 / 命中概率计算逻辑。
4. 修改赔率表。
5. 修改 package.json 或 lock 文件。
6. 新增依赖。
7. 修改部署配置。
8. 修改 Netlify 配置。
9. 删除文件。
10. 新增后端、数据库、登录。
11. 业务规则不明确。
12. build 失败且原因不明确。
13. git status 出现和任务无关的文件。
14. 任务范围明显扩大。

停止时必须返回：
- 卡在哪里
- 为什么不能继续
- 需要用户确认什么

## 执行任务要求

执行任何任务前，必须先确认：
- 任务目标
- 允许修改范围
- 禁止修改范围
- 必须运行的命令
- 成功标准
- 失败停止点

如果任务不清楚，不得自行猜测，应先提问或停止。
