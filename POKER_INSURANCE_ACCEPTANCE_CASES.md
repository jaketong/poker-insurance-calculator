# 扑克保险工具验收用例表 V1

## 一、文档目的

本文件用于固定扑克保险工具后续开发的验收标准。

以后只要修改以下内容，都必须对照本文件验收：

1. 德州扑克计算逻辑；
2. 奥马哈计算逻辑；
3. 短牌计算逻辑；
4. outs 计算；
5. 保险赔率表；
6. 结果页展示；
7. 复制结果文本；
8. 错误提示；
9. 手机截图体验。

本文件不是算法说明书，不负责定义新规则。
本文件只作为开发后的验收清单。

---

## 二、项目固定边界

| 编号 | 验收项 | 通过标准 |
|---|---|---|
| G-01 | 手机网页 | 一个链接打开即用，适合手机使用 |
| G-02 | 无登录 | 不出现登录、注册、账号系统 |
| G-03 | 无后端 | 不新增后端服务 |
| G-04 | 无数据库 | 不保存用户输入 |
| G-05 | 两家 All-in | 当前只支持两家 All-in |
| G-06 | 手动指定领先方 | 不自动判断领先方 |
| G-07 | 三种玩法独立 | 德州扑克、奥马哈、短牌互不混淆 |
| G-08 | 不做边池 | 不处理多人边池 |
| G-09 | 不做 GTO | 不加入 GTO 分析 |
| G-10 | 不做起手牌助手 | 不加入翻前起手牌推荐 |

## 三、德州扑克验收用例

| 编号 | 场景 | 重点验收 | 通过标准 |
|---|---|---|---|
| NLH-01 | 正常两家 All-in | 基础计算流程 | 能输出落后方反超 outs、概率、保险建议 |
| NLH-02 | 2 outs 场景 | 德州扑克赔率表 | 2 outs 对应赔率必须是 15 倍 |
| NLH-03 | 0 outs 场景 | 无反超可能 | 显示 0 outs，不应建议买保险 |
| NLH-04 | 多路径反超 | outs 去重 | 同一张牌不能重复计算 |
| NLH-05 | 重复牌输入 | 输入校验 | 必须提示重复牌，不能输出错误结果 |
| NLH-06 | 手牌不足 | 输入校验 | 必须提示补全 |
| NLH-07 | 公共牌异常 | 输入校验 | 公共牌数量异常时不能乱算 |

## 四、奥马哈验收用例

| 编号 | 场景 | 重点验收 | 通过标准 |
|---|---|---|---|
| PLO-01 | 正常两家 All-in | 基础计算流程 | 能输出 outs、概率、保险建议 |
| PLO-02 | 严格 2+3 规则 | 奥马哈核心规则 | 必须使用 2 张手牌 + 3 张公共牌 |
| PLO-03 | 只用 1 张手牌才成牌 | 规则陷阱 | 不能误判为有效成牌 |
| PLO-04 | 只用 3 张手牌才成牌 | 规则陷阱 | 不能误判为有效成牌 |
| PLO-05 | 高 outs 场景 | 去重与合法性 | outs 必须去重，且每张 out 都符合 2+3 |
| PLO-06 | 手牌不足 4 张 | 输入校验 | 必须提示补全 |
| PLO-07 | 重复牌输入 | 输入校验 | 必须提示重复牌 |
| PLO-08 | 德州逻辑误用 | 玩法隔离 | 奥马哈不能复用德州扑克任意 5 张逻辑 |

## 五、短牌验收用例

| 编号 | 场景 | 重点验收 | 通过标准 |
|---|---|---|---|
| SHORT-01 | 牌库范围 | 短牌只用 6-A | 2、3、4、5 必须视为非法牌 |
| SHORT-02 | A6789 | 特殊顺子 | A6789 必须识别为顺子 |
| SHORT-03 | 同花 vs 葫芦 | 短牌牌型大小 | 同花大于葫芦 |
| SHORT-04 | 顺子 vs 三条 | 短牌牌型大小 | 顺子大于三条 |
| SHORT-05 | 0 outs 场景 | 无反超可能 | 显示 0 outs |
| SHORT-06 | 重复牌输入 | 输入校验 | 必须提示重复牌 |
| SHORT-07 | 非法牌输入 | 输入校验 | 必须提示非法牌 |
| SHORT-08 | 玩法隔离 | 德州/奥马哈规则不能混入 | 短牌必须使用短牌牌库和牌型规则 |

## 六、赔率表验收用例

| 编号 | 场景 | 通过标准 |
|---|---|---|
| ODDS-01 | 德州扑克 2 outs | 必须显示 15 倍 |
| ODDS-02 | 0 outs | 不显示可购买保险赔率 |
| ODDS-03 | outs 不在赔率表 | 给出合理提示，不能乱套赔率 |
| ODDS-04 | 未经用户确认 | 不得修改赔率表 |
| ODDS-05 | 不同玩法赔率 | 德州、奥马哈、短牌不能误用错误赔率表 |

## 七、边界错误验收用例

| 编号 | 场景 | 通过标准 |
|---|---|---|
| ERR-01 | 重复牌 | 明确提示重复牌 |
| ERR-02 | 非法牌 | 明确提示非法牌 |
| ERR-03 | 手牌数量不足 | 不输出结果，提示补全 |
| ERR-04 | 公共牌数量不足 | 不输出结果，提示当前信息不足 |
| ERR-05 | 公共牌数量过多 | 不输出结果，提示牌数异常 |
| ERR-06 | 未选择领先方 | 不输出结果，提示先选择领先方 |
| ERR-07 | 无反超牌 | 显示 0 outs，不误导用户买保险 |
| ERR-08 | 同一张 out 多种路径命中 | 只算 1 次，不重复累加 |

## 十、后续使用规则

1. 文案或 UI 小改：至少检查结果页截图和复制结果用例；
2. 复制功能修改：必须检查 COPY 类用例；
3. 德州扑克算法修改：必须检查 NLH 类和 ODDS 类用例；
4. 奥马哈算法修改：必须检查 PLO 类用例，尤其是严格 2+3；
5. 短牌算法修改：必须检查 SHORT 类用例；
6. 赔率表修改：必须先获得用户明确确认；
7. 如果发现新 bug，修复后应补充一条对应验收用例。
